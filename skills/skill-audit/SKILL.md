# 元 Skill：Skill 质量审计

## 用途

对项目中的 skill 进行质量评估，输出评分和具体优化建议。

## 评分维度（共 8 维度，每项 12.5 分，满分 100）

### 1. 元数据完整性 (12.5 分)

检查 frontmatter 中的必需字段：

| 字段 | 分值 | 说明 |
|------|------|------|
| `name` | 4 | skill 唯一标识，格式：`custom:xxx` 或 `xxx` |
| `description` | 4.5 | 功能描述，应包含触发场景 |
| `argument-hint` | 4 | 参数提示，帮助 AI 理解输入格式 |

**评分标准：**
- 12.5：三个字段都存在且内容完整
- 8-12：缺少 argument-hint 或内容不完整
- 0-8：缺少多个字段

### 2. 触发词清晰度 (12.5 分)

评估 description 中的触发信息是否清晰：

| 等级 | 分值 | 标准 |
|------|------|------|
| 优秀 | 12.5 | 包含 `Use when` + 明确触发场景 + 触发词列表 |
| 良好 | 10 | 包含 `Use when` + 触发场景 |
| 一般 | 6 | 有描述但触发场景不明确 |
| 差 | 0-3 | 缺少触发信息 |

### 3. 结构规范性 (12.5 分)

检查内容结构是否规范：

| 检查项 | 分值 |
|--------|------|
| 有编号流程步骤 | 4 |
| 有代码/模板示例 | 4.5 |
| 有注意事项/说明 | 4 |

**评分标准：**
- 12.5：三项都有
- 8-12：缺少一项
- 0-8：结构混乱或缺少多项

### 4. 可操作性 (12.5 分)

评估 skill 是否可以直接执行：

| 检查项 | 分值 |
|--------|------|
| 有验收标准/检查清单 | 4.5 |
| 有明确的执行步骤 | 4 |
| 有预期输出说明 | 4 |

**评分标准：**
- 12.5：步骤明确 + 验收标准 + 预期输出
- 8-12：有步骤但缺少验收标准
- 0-8：描述模糊，难以执行

### 5. 内容质量 (12.5 分)

评估内容本身的质量：

| 检查项 | 说明 |
|--------|------|
| 准确性 | 描述准确，无歧义 |
| 实用性 | 内容有价值，非空洞 |
| 简洁性 | 无冗余，无重复 |

**评分标准：**
- 12.5：准确、实用、简洁
- 8-12：有少量冗余或可优化处
- 0-8：内容空洞或有明显错误

### 6. 可维护性 (12.5 分)

评估 skill 是否易于维护：

| 检查项 | 说明 |
|--------|------|
| 模块化 | 模板/配置是否独立 |
| 遵循最佳实践 | 格式规范、命名统一 |
| 易更新 | 修改是否影响其他部分 |

**评分标准：**
- 12.5：模块化、易维护
- 8-12：结构尚可但可优化
- 0-8：结构混乱，难以维护

### 7. 鲁棒性 (12.5 分)

评估边界情况和错误处理：

| 检查项 | 说明 |
|--------|------|
| 边界情况 | 是否考虑异常场景 |
| 错误处理 | 失败时的处理逻辑 |
| 容错能力 | 输入不完整时的表现 |

**评分标准：**
- 12.5：考虑全面，有容错设计
- 8-12：有基本容错
- 0-8：缺少错误处理

### 8. AI 友好度 (12.5 分)

评估对 AI 的友好程度：

| 检查项 | 说明 |
|--------|------|
| 简洁描述 | 指令清晰，无歧义 |
| 明确指令 | 使用祈使句，动作明确 |
| 易于理解 | 结构清晰，层次分明 |

**评分标准：**
- 12.5：AI 可直接理解并执行
- 8-12：基本可理解，有小优化空间
- 0-8：描述模糊，AI 难以理解

## 执行流程

### 审计单个 skill

1. 读取指定 skill 的 SKILL.md
2. 按 8 维度逐项评分
3. 为每个扣分项生成具体优化建议
4. 输出详细报告

### 审计全部 skills

1. 扫描 skills/ 目录下所有 skill
2. 逐个评估并记录分数
3. 生成汇总报告 + 优化建议
4. 计算平均分

## 状态图标

- `✓` 完美（满分）
- `⚠` 可优化（扣分 1-4）
- `✗` 需改进（扣分 >4）

## 验收标准

执行完成后必须满足：

- [ ] 输出了所有被审计 skill 的评分
- [ ] 每个维度都有明确的分数和状态图标
- [ ] 扣分项都有具体的优化建议
- [ ] 批量审计时显示了平均分
- [ ] 建议具体可执行，非泛泛而谈

## 注意事项

1. 评分应客观公正，基于实际内容而非主观判断
2. 优化建议应具体到行号或章节，便于用户定位
3. 对于新建的 skill，可适当放宽评分标准
4. 如果 skill 文件格式有误，先提示用户修复再评分
