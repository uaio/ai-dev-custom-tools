# 全局 orchestrator：新需求（任务放本项目skill目录下）

## 严格执行流程（必须100%遵守）

1. <thinking>...</thinking>
2. 理解用户需求 → 转换成专业主题（kebab-case）
3. **自动生成带时间标号的标题**：需求-YYYYMMDD-hhmm-专业主题（精确到分钟）
   示例：需求-20260314-0823-用户登录流程重构
4. 先展示标题，问用户“这个标题（含时间标号）是否确认？需要修改主题吗？”
5. 标题确认后，给出完整方案 + 风险说明，再次等待确认
6. 用户第二次回复“确认”“执行”“开始”“OK”等后：
   - 生成编号的任务列表（每条只写简述 + 涉及文件）
   - **立即在项目 .claude/skills/ 下创建主需求 skill 文件夹**：
     - 文件夹名：需求-YYYYMMDD-hhmm-专业主题
     - SKILL.md（轻量主索引）
     - tasks/ 子文件夹（每个任务一个独立 .md 文件）

# 需求完成与归档

## 完成检测

当所有任务状态变为”已完成”时，自动触发归档流程。

## 归档流程

1. **生成完整文档**：汇总所有任务到 `docs/requirements/需求-YYYYMMDD-hhmm-主题.md`
2. **展示摘要**：输出需求完成摘要，包含所有任务详情和最终结果
3. **确认归档**：
   ```
   需求”用户登录认证”已全部完成！

   ### 完成摘要
   - 任务数：5/5
   - 涉及文件：src/api/auth.ts, src/stores/auth.ts, ...
   - 完成时间：2026-03-17 16:30

   是否归档此需求？
   - 确认归档：移动到 docs/archived-requirements/ 并删除 skill
   - 暂不归档：保留 skill 以便后续查阅
   ```
4. **执行归档**（用户确认后）：
   - 移动文档到 `docs/archived-requirements/需求-YYYYMMDD-hhmm-主题.md`
   - **删除 skill 目录**：`rm -rf .claude/skills/需求-YYYYMMDD-hhmm-主题/`
5. **确认删除**：
   ```
   ✓ 已归档到 docs/archived-requirements/需求-20260317-1430-用户登录认证.md
   ✓ 已删除 skill: 需求-20260317-1430-用户登录认证
   ```

# 中断恢复规则

## 触发方式

- 用户说 “恢复需求-20260314-0823” 或 “继续需求-20260314-0823”
- 用户说 “恢复上次” 或 “继续上次”（自动匹配最近一个未完成的需求）
- 用户说 “恢复登录需求”（通过主题关键词匹配）

## 恢复执行流程

1. **定位需求目录**：根据时间标号或关键词找到 `.claude/skills/需求-XXX/` 目录
2. **读取主 SKILL.md**：获取任务列表和整体进度
3. **扫描 tasks/*.md**：按编号顺序检查每个任务状态
4. **生成断点摘要**：
   ```markdown
   ## 断点恢复摘要

   **需求**：需求-20260317-1430-用户登录认证
   **整体进度**：已完成 2/5

   ### 已完成任务
   - ✓ 任务01：创建 useAuthHook
   - ✓ 任务02：实现 jwt 双 token

   ### 断点位置
   - ⏸ 任务03：实现手机号登录 API（状态：进行中）
     - 中断原因：第三方短信服务接口超时
     - 涉及文件：src/api/auth.ts

   ### 待办任务
   - ○ 任务04：添加登录状态管理
   - ○ 任务05：编写单元测试

   ---
   是否从任务03继续？
   ```
5. **等待用户确认**：用户回复”继续””执行””OK”后从断点任务开始执行

## 任务状态定义

| 状态 | 标记 | 含义 |
|------|------|------|
| 待办 | ○ | 尚未开始 |
| 进行中 | ⏸ | 正在执行或中断暂停 |
| 已完成 | ✓ | 已通过验证 |
| 阻塞 | ⚠ | 因外部依赖暂停 |

## 更新记录规范

每次任务状态变更时，必须在任务文件中追加更新记录：
```markdown
## 更新记录
- 2026-03-14 08:23 创建
- 2026-03-14 09:15 开始执行
- 2026-03-14 10:30 中断 - 第三方短信服务接口超时
```
