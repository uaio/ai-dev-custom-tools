# Skill 质量评分系统设计

## 概述

为 open-skills 添加 `skills audit` 命令，实时分析所有 skill 的质量并给出评分和优化建议。

## 需求

- 无需持久化存储，实时计算评分
- 集成到 `install.sh` 中，作为 `skills audit` 命令
- 全面评估：元数据完整性 + 触发词清晰度 + 结构规范性 + 可操作性

## 评分维度

### 1. 元数据完整性 (25%)

检查 frontmatter 中的必需字段：

| 字段 | 分值 | 说明 |
|------|------|------|
| `name` | 8.3 | skill 唯一标识 |
| `description` | 8.3 | 功能描述 |
| `argument-hint` | 8.4 | 参数提示（可选但有加分） |

### 2. 触发词清晰度 (25%)

检查 description 中是否包含触发信息：

| 等级 | 分值 | 标准 |
|------|------|------|
| 优秀 | 25 | 包含 `Use when` 和明确触发词 |
| 中等 | 12 | 有描述但触发词不明确 |
| 差 | 0 | 缺少触发信息 |

### 3. 结构规范性 (25%)

检查内容结构：

| 检查项 | 分值 |
|--------|------|
| 有编号流程步骤 | 8.3 |
| 有代码/模板示例 | 8.3 |
| 有注意事项/说明 | 8.4 |

### 4. 可操作性 (25%)

评估执行可行性：

| 等级 | 分值 | 标准 |
|------|------|------|
| 强 | 25 | 步骤明确、有验收标准、可直接执行 |
| 中 | 15 | 有步骤但不够具体 |
| 弱 | 5 | 描述模糊，难以执行 |

## 命令接口

```bash
skills audit              # 审计所有 skill
skills audit <skill-name> # 审计单个 skill
```

## 实现方案

在 `install.sh` 中添加以下函数：

1. `audit_skill()` - 审计单个 skill
2. `audit_all_skills()` - 审计所有 skill
3. `show_audit_help()` - 显示帮助

在 `main()` 的 case 语句中添加 `audit` 分支。
